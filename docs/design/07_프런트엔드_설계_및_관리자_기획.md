# [상세 설계] 07_프런트엔드_UX_UI_및_관리자_설계

## 1. 디자인 철학: "Agent-First, Data-Driven"
문서 전달 중심의 '블로그' 형태를 탈피하고, GPT/Gemini와 같은 **대화형 인터페이스(LUI, Language User Interface)**를 기반으로 효율적인 업무 관리 기능을 조화시킨 심플한 디자인을 지향합니다.

---

## 2. 사용자 인터페이스 (User UI)

### 2.1 메인 레이아웃 (Agent Chat Hub)
- **중앙 캔버스**: 에이전트와 대화하는 채팅 인터페이스. 텍스트뿐만 아니라 코드 스니펫, 그래프, 문서 미리보기가 카드 형태로 렌더링됨.
- **좌측 사이드바 (업무 중심)**:
  - **오늘의 일정 (Calendar/ToDo)**: 회사 생활에 필수적인 일 단위 태스크 트래킹.
  - **진행 중인 실습**: 현재 수행 중인 Java 실습 항목과 달성도.
- **우측 사이드바 (Context)**: 현재 대화에서 참조 중인 `rules.md`나 관련 가이드 문서의 요약 정보 노출.

### 2.2 인터랙션 요소
- **심플한 입력창**: 자연어 입력뿐만 아니라 `/` 명령어를 통해 "코드 검토", "일정 등록" 등의 빠른 액션 지원.
- **미니멀한 애니메이션**: 에이전트의 사고 과정(Thought Process)을 시각적으로 보여주는 로딩 UI.

---

## 3. 관리자 센터 (Admin Center)

관리자가 에이전트의 '성격'과 '규칙'을 제어하는 제어실 역할을 합니다.

### 3.1 페르소나 및 룰 관리 (Persona Setup)
- **에이전트 페르소나 설정**: 
  - (예: 엄격한 시니어 개발자, 친절한 동기, 효율 중심의 테크 리드) 말투와 강조할 가치를 설정.
- **금칙어 및 필터 관리**: 
  - 비속어, 사내 보안 키워드, 혹은 "모르겠다"는 회피성 답변 방지 로직 설정.
- **Rules Sync**: `rules.md` 수정 시 에이전트가 반영할 우선순위 조정.

### 3.2 대시보드 (Admin Dashboard)
- **온보딩 통계**: 신입 사원별 실습 통과율, 가장 많이 질문한 키워드(Pain Point) 시각화.
- **에이전트 정확도**: 사용자의 "도움이 되었나요?" 피드백 지표 관리.
- **토큰 사용량/비용**: LLM API 호출 통계 관리.

---

## 4. 페이지 구성 정보
1. **Chat Home**: 에이전트와의 메인 인터페이스.
2. **Work Planner**: 일 단위/주 단위 업무 관리 캘린더 뷰.
3. **Admin Settings**: 페르소나, 금칙어, 룰 관리 페이지.
4. **Insight Insight (Admin Only)**: 신입 사원 성장 지표 및 에이전트 성능 대시보드.

---

## 5. 변경되는 기술 사양
- **LUI 컴포넌트**: `react-chat-ui` 또는 커스텀 프로포지션(Tailwind) 기반의 대화방 구현.
- **Charts**: 대시보드 시각화를 위한 `Recharts` 또는 `Chart.js` 추가.
- **Real-time**: 관리자 설정 변경 시 에이전트에 즉시 반영하기 위한 상태 소켓 연동 제안.
