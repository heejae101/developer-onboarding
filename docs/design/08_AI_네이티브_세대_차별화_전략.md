# [상세 설계] 08_AI_네이티브_세대_차별화_전략

본 문서는 이미 Copilot, ChatGPT 등 AI 도구에 익숙한 2026년 신입 개발자를 위한 차별화 전략을 정의합니다.

---

## 🎯 문제 인식: AI 기본 장착 세대

> [!WARNING] 기존 온보딩의 한계
> 2026년 신입 개발자는 이미 AI 코딩 어시스턴트를 능숙하게 사용합니다.
> 단순히 "AI가 도와줍니다"는 더 이상 차별화 포인트가 아닙니다.

### 그들이 이미 할 수 있는 것
- Copilot으로 코드 자동 완성
- ChatGPT로 에러 메시지 해석
- Claude로 코드 리뷰 요청

### 그들이 할 수 없는 것 (= 우리의 기회)
- **우리 팀의 도메인**을 이해하고 적용하기
- **우리 프로젝트의 히스토리**를 파악하기
- **우리 회사의 레거시 코드**를 빠르게 읽기
- **제품 감각**을 키우고 "무엇을 만들지" 결정하기

---

## 🚀 차별화 전략 (4가지 핵심 무기)

### 1. 도메인 특화 학습 경로 (Domain-Specific Learning Path)

**일반 AI**: "Spring Security란 인증과 인가를 처리하는 프레임워크입니다."  
**우리 AI**: "우리 팀은 JWT + Redis 세션 이중화 방식을 씁니다. 이유는 [링크]. 관련 코드는 `AuthService.java` 참고."

| 기능 | 설명 |
|:---|:---|
| **회사 전용 RAG** | 사내 문서, Confluence, Slack 히스토리를 학습한 지식 베이스 |
| **도메인 용어 사전** | "지적도", "속성 테이블" 등 GIS 업계 용어 자동 설명 |
| **프로젝트별 컨텍스트** | 현재 진행 중인 프로젝트의 요구사항과 제약조건 자동 주입 |

---

### 2. 레거시 코드 온보딩 (Legacy Code Navigator)

**일반 AI**: "이 코드는 Singleton 패턴입니다."  
**우리 AI**: "이 Singleton은 2019년에 메모리 누수 버그 때문에 도입됐어요. 당시 PR은 [#1234]. 지금은 더 좋은 방법이 있지만, 연관된 코드가 12개라 리팩토링 우선순위는 낮습니다."

| 기능 | 설명 |
|:---|:---|
| **Git Blame 스토리텔링** | "왜 이렇게 짰는지" 커밋 히스토리 기반 설명 |
| **의존성 지도** | 특정 클래스를 수정하면 영향받는 범위 시각화 |
| **리팩토링 우선순위** | 기술 부채 해결 시 ROI가 높은 순서 제안 |

---

### 3. 제품 감각 교육 (Product Sense Training)

> [!IMPORTANT] 프로덕트 엔지니어 육성
> 코딩 속도는 AI가 올려주지만, "무엇을 만들지" 결정하는 능력은 여전히 인간의 영역입니다.

| 기능 | 설명 |
|:---|:---|
| **일일 회고 질문** | "오늘 작업이 사용자에게 어떤 가치를 줬나요?" |
| **사용자 피드백 연결** | 내가 만든 기능에 대한 실제 사용자 반응 노출 |
| **의사결정 로그** | "왜 A 대신 B를 선택했는지" 기록하고 회고하기 |
| **미니 PM 미션** | 주 1회 "다음에 뭘 만들면 좋을지" 제안 작성 |

---

### 4. 자율 에이전트 구축 경험 (Build Your Own Agent)

**일반 AI 사용자**: 프롬프트만 입력하고 결과를 받음  
**우리 플랫폼 사용자**: 자신만의 에이전트를 직접 설계하고 배포

| 기능 | 설명 |
|:---|:---|
| **에이전트 템플릿** | 코드 리뷰, 문서 생성, 테스트 자동화 등 템플릿 제공 |
| **LangGraph 시각화** | 노드와 엣지를 시각적으로 설계하는 빌더 |
| **Git Worktree 실습** | 병렬 에이전트 실행을 직접 체험 |
| **자율 루프 모니터링** | Think-Act-Observe 사이클 실시간 관찰 |

## 🎯 기대 효과

- **신입 개발자**: "여기서만 배울 수 있는 것"이 있어서 초기 적응 속도와 팀 소속감 상승
- **팀 전체**: 도메인 지식과 레거시 히스토리가 AI를 통해 자동 전수되어 지식 유실 방지
- **조직 경쟁력**: 단순 코딩 속도를 넘어 "제품을 만드는 엔지니어"를 체계적으로 양성
