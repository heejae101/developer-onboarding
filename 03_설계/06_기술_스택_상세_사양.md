# [상세 설계] 06_기술_스택_상세_사양

본 프로젝트의 안정성과 호환성을 위해 최신 LTS(Long Term Support) 및 각 생태계에서 검증된 기술 버전을 사용합니다.

---

## 1. Backend (Java)
회사의 주력 언어인 Java 환경을 반영합니다.

- **Language**: Java 17 (LTS) - 현대 자바의 표준이자 가장 안정적인 버전
- **Framework**: Spring Boot 3.2.x
- **Build Tool**: Gradle 8.x
- **Database**: 
  - PostgreSQL 15.x (LangGraph 체크포인트 및 메인 데이터 관리용)
  - Redis 7.x (선택 사항: 작업 캐싱 및 실시간 알림용)
- **Main Libraries**:
  - Spring Data JPA
  - Spring Security (JWT 기반)
  - Querydsl 5.x (복잡한 쿼리 처리용)

---

## 2. AI Agent Layer (Python)
LangGraph 및 AI 로직을 위한 전용 사이드카 서비스입니다.

- **Language**: Python 3.11+
- **Framework**: FastAPI (MCP 서버 구현용)
- **AI Framework**:
  - LangChain 0.2.x
  - LangGraph 0.1.x (Stateful Multi-agent 로직 핵심)
- **Integration**:
  - `mcp` (Model Context Protocol SDK)
  - `openai` 또는 `anthropic` SDK (LLM 연동)
  - `pandas`, `openpyxl` (Excel 파싱용)

---

## 3. Frontend (Web)
사용자 친화적인 인터페이스를 위한 모던 웹 기술 스택입니다.

- **Framework**: React 18+
- **Build Tool**: Vite 5.x (빠른 개발 생산성)
- **Styling**: Tailwind CSS 3.x (Glassmorphism 및 디자인 시스템 구축)
- **Main Libraries**:
  - `lucide-react` (아이콘)
  - `react-query` (서버 상태 관리)
  - `zustand` (클라이언트 상태 관리)
  - `openlayers` (GIS 지도 실습용)

---

## 4. Infrastructure & DevOps
- **Container**: Docker & Docker Compose (로컬 개발 환경 통합)
- **VCS**: Git (GitHub 또는 GitLab)
- **Communication**: Slack Webhook (워크플로우 알림 연동)

---

## 5. 설계 결정 사유 (Summary)
- **Next-gen Compliance**: MCP 프로토콜을 사용하여 향후 어떤 LLM(Claude, GPT, Gemini 등)이 나와도 쉽게 교체 가능한 유연한 구조를 지향합니다.
- **Java 17 선택**: Java 8/11보다 성능이 우수하고, Spring Boot 3.x의 최소 요구사항을 만족하기 위함입니다.
