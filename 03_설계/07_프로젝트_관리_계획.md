# Part 1: 성공 지표 및 OKR

## 📊 핵심 성과 지표 (KPIs)

| 지표 | 기준선 (AS-IS) | 목표 (TO-BE) | 측정 방법 |
|:---|:---|:---|:---|
| **첫 PR까지 소요일** | 14일 | 7일 (50% 단축) | 입사일 ~ 첫 Merge PR 날짜 |
| **시니어 온보딩 투입 시간** | 주 10시간 | 주 3시간 (70% 감소) | 멘토링 시간 로그 |
| **수동 엑셀 업데이트** | 월 20회 | 월 5회 이하 (75% 감소) | 파일 업로드 로그 |
| **룰 위반 PR 리뷰 지적** | PR당 3건 | PR당 0.5건 이하 (83% 감소) | 코드 리뷰 코멘트 분석 |
| **알림 자동 발송 성공률** | N/A | 99% 이상 | SMTP/Slack 발송 성공률 |

## 🎯 OKR (분기별 목표)

### Q1: MVP 출시 및 초기 검증

| Objective | Key Results |
|:---|:---|
| **O1: MVP 성공 출시** | KR1: 일일 업무 기록 기능 완성 / KR2: LangGraph 기본 라우팅 구현 / KR3: WebSocket 스트리밍 동작 확인 |
| **O2: 자동화 검증** | KR1: 스케줄러 알림(Slack/이메일) 구현 / KR2: 엑셀 자동 분석 파이프라인 동작 확인 |

### Q2: 고도화 및 확산

| Objective | Key Results |
|:---|:---|
| **O3: 에이전트 강화** | KR1: Git Worktree 병렬 에이전트 2개 동시 실행 / KR2: Ollama 응답 시간 3초 이내 유지 |
| **O4: 팀 확산** | KR1: 팀원 3명 활성 사용자 확보 / KR2: 첫 PR 소요일 50% 단축 달성 |

# Part 2: 리스크 분석 및 대안

## 🚨 주요 리스크 및 Plan B

| 등급 | 리스크 | Plan B |
|:---|:---|:---|

| 🔴 **Critical** | 1인 개발 일정 지연 | MVP 축소 (P1+P2만 우선, P3~P7 2차 릴리즈) |
| 🟠 **High** | LangGraph 학습 곡선 | 단순 상태 머신(if-else 라우팅) 대체 |

| 🟡 **Medium** | Git Worktree 충돌 | 에이전트별 작업 영역 분리 |
