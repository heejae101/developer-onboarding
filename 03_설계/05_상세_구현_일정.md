# [상세 설계] 05_상세_구현_일정 (WBS)

본 일정은 1인 개발 기준으로, 핵심 기능을 빠르게 검증하고 고도화하는 4주 과정의 상세 태스크입니다.

---

## 📅 1주차: 인프라 및 핵심 기초 기능 (P1 중심)
- **Day 1**: 개발 환경 구축
  - Java Spring Boot 프로젝트 구조 생성 및 DB(PostgreSQL) 연동
  - React/Vite 프런트엔드 프로젝트 초기화 및 레이아웃(Sidebar/Main) 구성
- **Day 2**: 사용자 및 권한 기초 구현
  - JWT 기반 인증 시스템 및 회원가입/로그인 API 개발
  - 기본 권한(Role) 체계 DB 반영
- **Day 3-4**: 일일 업무 기록 기능 (P1)
  - 업무 기록(Daily Work) CRUD API 및 DB 설계 반영
  - 달력(Calendar) 라이브러리를 활용한 대시보드 UI 구현
- **Day 5**: 지능형 목표 관리 레이어
  - 목표 달성도 계산 로직 개발
  - 상태값에 따른 시각적 피드백(잔디 심기 등) 구현

---

## 📅 2주차: AI 에이전트 및 MCP 연동 (P2, P3 중심)
- **Day 6**: Python MCP 서버 초기화 (P3)
  - FastAPI 기반 MCP 서버 뼈대 구성
  - 로컬 파일 시스템 리더/라이터 MCP 도구 구현
- **Day 7**: LangGraph 환경 및 상태 설계 (P2)
  - LangGraph Python 패키지 설정 및 기본 Graph State 정의
  - 간단한 인사 및 대화 맥락 유지(Memory) 테스트
- **Day 8**: 지능형 라우팅 로직 개발
  - Router Node 구현 (질문 vs 실행 vs 검증 판별)
  - Java 백엔드와 MCP 서버 간의 JSON-RPC 통신 레이어 연동
- **Day 9-10**: MCP 기반 Excel 연동 (P3)
  - `pandas/openpyxl`을 활용한 엑셀 데이터 파싱 도구 구현
  - 엑셀 데이터를 DB로 마이그레이션하거나 AI 컨텍스트로 주입하는 노드 개발

---

## 📅 3주차: 프로젝트 룰 및 워크플로우 자동화 (P4, P5 중심)
- **Day 11**: 프로젝트 룰 엔진 구축 (P5)
  - `rules.md` 파싱 및 카테고리별 DB 저장 로직 개발
  - RAG(검색) 노드를 통한 룰 탐색 도구 연동
- **Day 12**: 룰 가디언(Verification) 구현
  - 코드/문서 입력에 대한 룰 준수 여부 판별 노드 상세화
  - 수정 제안(Suggestion) 프롬프트 튜닝
- **Day 13-14**: n8n 스타일 워크플로우 엔진 (P4)
  - 트리거(Trigger) 시스템 구축 (스케줄러, 파일 변경 감지)
  - 노드 간 데이터 전달 및 체이닝 로직 구현
- **Day 15**: 알림 및 피드백 시스템
  - 워크플로우 결과에 대한 Slack/Web 알림 기능 연동

## 📅 4주차: 실습 영역 및 고도화 (P6, P7 중심)
- **Day 16-17**: GIS 지도 실습 영역 (P6)
  - OpenLayers/Cesium 라이브러리 연동 및 샘플 데이터 로딩
  - 지도 조작 및 위치 기반 데이터 시각화 예제 구현
- **Day 18**: 지식 공유 게시판 (P7)
  - MD 뷰어 기능을 포함한 게시판 목록/상세 구현
- **Day 19**: 통합 테스트 및 QA
  - 전체 사용자 시나리오(온보딩 flow) 엔드 투 엔드 테스트
  - 성능 최적화 및 보안 취약점 점검
- **Day 20**: 최종 문서화 및 시연 준비
  - 신입 사원용 사용 가이드 작성 및 배포 패키징 구성
