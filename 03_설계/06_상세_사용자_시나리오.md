본 문서는 P1~P7 각 기능에 대한 구체적인 사용자 스토리를 정의합니다.

|:---|:---|:---|

## 📖 시나리오 1: 신입 개발자의 첫 출근 (P1 - 일일 업무 기록)

**As a** 김신입  
**I want to** 오늘 해야 할 일과 학습 목표를 대시보드에서 확인하고 기록하고 싶다  
**So that** 내가 무엇을 배우고 있는지 트래킹하고, 팀장에게 진행 상황을 공유할 수 있다

### 상세 흐름
1. 김신입이 아침 9시에 플랫폼에 로그인한다.
2. **대시보드**에서 오늘의 학습 목표(예: "Spring Security 기초 이해")를 확인한다.
3. 오전 업무 후, 체크박스를 클릭하여 완료 표시를 한다.
4. **AI 에이전트**가 "오늘 Spring Security에서 가장 어려웠던 부분이 뭐였어?"라고 질문한다.
5. 김신입이 텍스트로 회고를 작성하면, AI가 요약하여 **일일 리포트**를 생성한다.
6. 퇴근 시점에 **스케줄러**가 자동으로 박매니저에게 **Slack/이메일**로 일일 리포트를 발송한다.

---

## 📖 시나리오 2: 모호한 질문에 대한 지능형 라우팅 (P2 - LangGraph 에이전트)

**As a** 김신입  
**I want to** "이거 어떻게 해요?"처럼 모호한 질문을 해도 AI가 알아서 처리해줬으면 좋겠다  
**So that** 시니어에게 매번 물어보지 않고도 스스로 해결할 수 있다

### 상세 흐름
1. 김신입이 채팅창에 "우리 팀 API 만드는 규칙이 뭐야?"라고 입력한다.
2. **LangGraph 라우터**가 이 질문을 분석하여 "룰 검색" 노드로 라우팅한다.
3. **RAG 노드**가 `rules.md`에서 관련 규칙을 검색하여 응답한다.
4. 만약 "이 규칙대로 Controller 코드 짜줘"라고 추가 요청하면, **코드 생성 노드**로 라우팅된다.
5. 생성된 코드는 **WebSocket**을 통해 실시간으로 스트리밍되어 화면에 표시된다.

---

## 📖 시나리오 3: 엑셀 체크리스트 자동화 (P3 - MCP 데이터 통합)

**As a** 이시니어  
**I want to** 매주 반복하는 엑셀 업데이트를 자동화하고 싶다  
**So that** 단순 반복 업무에서 벗어나 핵심 개발에 집중할 수 있다

### 상세 흐름
1. 이시니어가 주간 업무 체크리스트 엑셀 파일을 업로드한다.
2. **MCP 도구**가 엑셀을 파싱하여 미완료 항목을 추출한다.
3. AI가 "미완료 항목 3개: A, B, C. 우선순위 제안: B > A > C"라고 분석 결과를 제시한다.
4. 이시니어가 승인하면, **워크플로우**가 자동으로 Jira 티켓을 업데이트하고 **Slack 알림**을 발송한다.

---

## 📖 시나리오 4: 트리거 기반 업무 자동화 (P4 - 워크플로우)

**As a** 박매니저  
**I want to** 특정 조건이 되면 자동으로 알림이 가도록 설정하고 싶다  
**So that** 일일이 체크하지 않아도 중요한 이벤트를 놓치지 않는다

### 상세 흐름
1. 박매니저가 워크플로우 빌더에서 새 자동화를 생성한다.
   - **트리거**: "매일 오후 6시" (스케줄러)
   - **조건**: "오늘 일일 업무 기록 미작성자가 있으면"
   - **액션**: "해당 팀원에게 **SMTP 이메일** 리마인더 발송"
2. 저장하면 **Spring Scheduler**가 매일 해당 시간에 체크한다.
3. 조건 충족 시 **JavaMailSender**가 자동으로 이메일을 발송한다.

---

## 📖 시나리오 5: 룰 위반 실시간 탐지 (P5 - Rule Guardian)

**As a** 김신입  
**I want to** 내가 짠 코드가 팀 규칙에 맞는지 실시간으로 알고 싶다  
**So that** PR 리뷰에서 지적받기 전에 미리 수정할 수 있다

### 상세 흐름
1. 김신입이 코드 에디터에서 새 Controller를 작성한다.
2. **Rule Guardian 에이전트**가 실시간으로 코드를 분석한다.
3. `@RequestMapping` 대신 `@GetMapping`을 사용하라는 팀 규칙 위반을 탐지한다.
4. **WebSocket**을 통해 즉시 경고 메시지가 에디터 하단에 표시된다.
5. "자동 수정" 버튼을 누르면 AI가 규칙에 맞게 코드를 수정해준다.

---

## 📖 시나리오 6: GIS 지도 실습 (P6 - 지도 예제)

**As a** 김신입  
**I want to** 우리 팀에서 사용하는 OpenLayers/Cesium을 직접 실습해보고 싶다  
**So that** 실무 투입 전에 GIS 기술 스택에 익숙해질 수 있다

### 상세 흐름
1. 김신입이 "지도 실습" 메뉴로 이동한다.
2. **샘플 데이터**(예: 서울시 건물 레이어)를 선택하여 지도에 로딩한다.
3. 줌/패닝/레이어 토글 등 기본 조작을 실습한다.
4. AI가 "이 레이어의 데이터 구조를 보여줄까?"라고 제안한다.
5. 승인하면 GeoJSON 구조를 설명해주고, 관련 API 문서 링크를 제공한다.

---

## 📖 시나리오 7: 지식 공유 및 아카이빙 (P7 - 게시판)

**As a** 이시니어  
**I want to** 내가 삽질한 내용을 팀 위키에 기록하고 싶다  
**So that** 동료들이 같은 실수를 반복하지 않도록 도울 수 있다

### 상세 흐름
1. 이시니어가 "지식 공유" 메뉴에서 "새 글 작성"을 클릭한다.
2. 마크다운 에디터에서 트러블슈팅 내용을 작성한다.
3. AI가 "이 글에 적합한 태그를 추천해줄까?"라고 제안한다.
4. 태그를 추가하고 게시하면, 관련 키워드로 검색 가능한 상태가 된다.
5. 추후 김신입이 비슷한 오류를 겪으면, **RAG 검색**을 통해 이 글이 추천된다.
