# MVP 개발 및 검증 전략 (2주 집중형)

최소 기능 제품(MVP)의 목표는 **"AI 에이전트가 우리 팀의 프로젝트 룰을 바탕으로 가이드를 해줄 수 있는가?"**를 2주 안에 검증하는 것입니다.

---

## 1. MVP 핵심 기능 정의 (Must-Have)
추상적인 기능을 걷어내고 다음 3가지에 기술력을 집중합니다.

1.  **Rule-based Chat Agent**: `rules.md`를 읽어 답변하는 LangGraph 에이전트.
2.  **Java Code Practice**: 신입 사원이 Java 코드를 입력하면 AI가 룰 위반 여부를 검사해 주는 화면.
3.  **Basic Dashboard**: 일일 업무 기록과 실습 완료 상태를 보여주는 간단한 UI.

> [!NOTE] 제외 항목 (Nice-to-Have)
> - 지도 실습 영역, 복잡한 워크플로우 시각화 엔진, 고도화된 스케줄링 등.

---

## 2. MVP용 기술 스택 최적화
오버 엔지니어링을 최소화하기 위해 다음 환경으로 시작합니다.

- **Frontend**: React (Vite) + Tailwind CSS
- **Backend**: Java Spring Boot (포스트그레스 연동)
- **AI**: Python LangGraph (MCP 서비스 형태로 가장 가볍게 구현)

---

## 3. MVP 2주 개발 로드맵 (WBS)

### 📅 1주차: 인터페이스 및 AI 기초
- **목표**: 상호작용 가능한 최소한의 에이전트 환경 구축.
- **Day 1-2**: Java 백엔드 환경 설정 및 '프로젝트 룰' DB화.
- **Day 3-4**: 랭그래프 기반의 RAG(문서 검색) 에이전트 구현 (Python).
- **Day 5**: 프런트엔드 채팅 UI 및 Java 백엔드-AI 연동.

### 📅 2주차: 기능 통합 및 검증
- **목표**: 실제 사용자 시나리오 완성과 검증.
- **Day 6-7**: Java 코드 실습 화면 및 AI 코드 리뷰 기능 연동.
- **Day 8-9**: 업무 기록(ToDo 스타일) 및 대시보드 기초 완성.
- **Day 10**: 실제 샘플 규칙(`rules.md`) 입력 후 신입 사원 시나리오 테스트.

---

## 4. MVP 성공 판정 기준
- 질문을 던졌을 때 `rules.md`의 내용을 정확히 참조하여 답변하는가?
- Java 코드를 제출했을 때 컨벤션 위반 사항을 3초 이내에 찾아내는가?
- 위 과정이 하나의 웹 대시보드 안에서 끊김 없이 일어나는가?
